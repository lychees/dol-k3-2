# Uncharted Waters — three.js voyage · Design Document

## 1. Overview

A single-page browser game recreating *Uncharted Waters 2* (Koei, 1993) in the
browser with **three.js**, using data and media assets extracted from the
[uw2ol](https://github.com/timewarpsgh/uw2ol) project. The game is a static
site (no build step, no backend) deployed on GitHub Pages.

Core loop: **sail → trade → outfit your fleet → fight pirates → explore ashore
→ discover the world.**

- No build tooling: plain ES modules + an import map (`three` from `game/lib/`)
- No server logic: all state lives in the browser (localStorage)
- All game data (maps, prices, ships, mates, music) exported from uw2ol's
  Python data files into JSON/binary assets by `tools/` scripts

## 2. Architecture

```
game/
  index.html        shells, panels, HUD, import map
  main.js           the whole game (~2.5k lines, sections below)
  lib/three.module.js
  assets/           generated by tools/ from the uw2ol repo
tools/              asset pipeline (Python): prepare_assets.py, prepare_port_assets.py
tests/              Playwright suites (tests/screenshots/ for captures)
```

### 2.1 main.js sections (top to bottom)

| Section | Responsibility |
|---|---|
| Constants | map sizes, speeds, walkable/sailable tile id sets |
| Boot | parallel asset loading (fetch JSON/bin + textures) |
| `makeTilemapMesh` | tilemap shader factory (world map & port maps) |
| `makeSprite` / frame helpers | flat sprite quads & direction-frame math |
| Sea scene | world map plane, ship, port markers |
| Port scene | person sprite, port map, NPCs |
| Panel manager | `PANELS` registry: open/close/query all UI panels |
| Dialog | NPC talk lines + agent/old-man tips |
| Port NPCs | wanderers + static door NPCs |
| Player state `P` | save/load, fleet, cargo, crew, mates, equipment, hero |
| Buildings | per-building action menus (12 types) |
| Market / Shipyard / Outfit / Mates panels | datatable UIs |
| Land mode | ashore walking, encounters, DQ battles |
| Naval battles | pirates, cannonballs, boarding, captures |
| Music | region-based port/sea/building tracks + sfx |
| HUD / input / game loop | tick() drives everything |

### 2.2 Rendering: the tilemap shader

Both the world map (2160×1080) and port maps (96×96) are **one
`PlaneGeometry` each**. The tile id grid lives in a `DataTexture`
(RedFormat/UnsignedByte); a GLSL3 fragment shader looks up the tile id per
pixel and samples the right 16×16px cell from the tileset atlas.

- Exact texel-center sampling keeps close-ups pixel-crisp
- `textureGrad` with continuous derivatives → correct mip level +
  anisotropy in the distance (no moiré on the EGA dither patterns)
- Day/night: two tileset textures (phase A/B) blended in-shader with a
  cross-fade factor
- Port maps use the same shader with different tileset dimensions
  (16×15 grid) and +1-shifted data (PORTMAP bytes are 0-based)

Sprites are flat quads (`makeSprite`) with `repeat`/`offset` UV windows into
their sprite sheets; walking/sailing animation is 2 frames per direction.

### 2.3 Scenes & state machine

`scene ∈ { 'sea', 'port', 'land' }`, one `tick()` loop branches per scene:

- **sea**: ship movement with coast sliding, port/village hints, pirates &
  battle, port discovery, day/night
- **port**: person walking with tile collision, buildings, NPCs + dialog,
  port minimap
- **land**: hero walking on land tiles, random encounters, discovery by
  proximity

Panels (market, shipyard, mates, outfit, menu, dev, dialog) are managed by a
single registry (`openPanel/closePanel/anyPanelOpen/closeTopPanel`), which
drives Esc/E key routing and movement locking (`panelOpen`).

### 2.4 Persistence

`P` (player state) is serialized to `localStorage` on every mutation via
`save()`: gold, fame, fleet, cargo (+cost), bank, crew, mates, cabins,
equipment, hero, quests, discoveries, ports, days, character, devSpeed.
Migrations run at load for older save shapes (3-tier ships → named ships →
fleet).

## 3. Game systems

### 3.1 Sailing & time

- 180 s = 1 in-game day; new-day effects: provisions −8 (× boatswain),
  fatigue +12 (× figurehead × surgeon), hull −1, bank interest +2%
- Speed = slowest fleet ship × sails × navigator × penalties
  (starving/exhausted/under-crewed)
- Day/night phases (dawn/day/dusk/night) cross-fade the original tilesets

### 3.2 Economy

- 13 regional price tables (~46 goods each, `[buy, sell]`) + 67 port
  specialties, straight from uw2ol data
- Cargo tracked with purchase cost → per-unit profit/loss shown when selling
- Bank: 2% daily interest; accountant cabin raises sell prices

### 3.3 Fleet & crew

- Up to 5 ships (22 real types): cargo=sum, guns=sum, speed=min, crew
  limits=sums; flagship takes battle damage
- Sailors hired at bars; under minimum crew → speed & broadside penalties
- 50 named mates (uw2ol data) hired in bars; 6 cabin slots with stat-driven
  bonuses (Navigator/Gunner/Accountant/Lookout/Surgeon/Boatswain)
- Ship equipment: sails & cannons (3 tiers), ram, figurehead, boarding
  planks, armor plating

### 3.4 Naval combat

- Pirates spawn over time, chase, engage on contact
- Real-time broadsides (SPACE, 2 s cooldown, homing cannonballs)
- Dual health: hull (sinks) + crew (captures); cannonballs cause crew
  casualties (30% enemies / 10% player)
- Boarding melee (B, <2.5 tiles): swordplay-driven attrition; planks +25%;
  capture joins the fleet or prize money; aggressive pirates board after an
  8 s grace period when 1.5× your crew

### 3.5 Ports & NPCs

- 96×96 tile ports from original PORTMAP data; 12 building types with real
  functions (market, bar, dry dock/shipyard, harbor, inn, palace, job house,
  MSC, bank, item shop, church, fortune house)
- NPCs: 4 wanderers (random walk) + static dog/old man/agent/guard at
  doors; E to talk (uw2ol's lines; agent gives real trade tips, old man
  gives discovery coordinates); hidden & non-interactable at night
- Quests: MSC discovery research, job house delivery, palace fame milestones

### 3.6 Land expeditions

- L to go ashore / re-board; walk land tiles of the world map
- Random encounters → turn-based JRPG battles (Attack/Balm/Run) with the
  hero + up to 3 mates; hero levels, equipment (weapons/armor/balms)
- Discoveries trigger by walking within 1.5 tiles of a site

## 4. Asset pipeline

`tools/prepare_*.py` (require a local uw2ol clone) export:

| Asset | Source | Format |
|---|---|---|
| `world_map.bin` | world_map piddle | 2160×1080 bytes, 1-based tile ids |
| `portmaps.bin` | PORTMAP.000-100 | 101×(96×96) bytes, 0-based |
| `tiles_*.png`, `portchips/` | original tilesets | 16px tiles |
| `ports.json`, `port_meta.json` | hash_ports_meta_data | coords, buildings, tileset, region |
| `goods.json` | hash_markets_price_details + hash_special_goods | 13 regional tables + specialties |
| `ships.json` | hash_ship_name_to_attributes | 22 ships: power/capacity/guns/crew/price |
| `mates.json`, `figures.png` | hash_mates | 50 mates + 65×81 portraits |
| `villages.json`, `discoveries.png` | hash_villages | 98 sites + 49px art cells |
| `music/`, `sounds/` | uw2ol sounds | regional BGM + effects |

## 5. Music & audio

- Port music: 6 capital themes > regional towns > China/Japan/Oceania by id
  > `port.ogg` fallback (uw2ol's mapping, region exported per port)
- Sea music on set sail by the port's region; building themes for
  bar/church/palace; `battle.ogg` for land battles; sfx channel for
  effects (discover, wave, shoot, explosion, engage)

## 6. Testing

`tests/` holds Playwright suites per feature area (buildings, battle,
boarding, fleet, mates, land, npc, dialog, dev, port, bgm). They run
headless Chromium against a local static server and drive the game through
`window.UW` debug hooks (teleport, spawnPirate, evaluate state, click real
UI). Screenshots land in `tests/screenshots/`.

Notes: the headless SwiftShader environment is slow/unstable (unthrottled
rAF, occasional context loss), so tests use poll-based waits and
deterministic debug hooks (`debugHit`, `hurtEnemy`, `setNoAutoSpawn`)
instead of wall-clock assertions.

## 7. Deployment

GitHub Pages serves the repo root; `/game/` is the game. `main.js` is
cache-busted with a `?v=` query on updates.
